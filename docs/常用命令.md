# SmartWheel 常用命令

本文档收集日常调试和现场使用最频繁的命令。每条命令都附带前置条件和注意事项，
请阅读后再执行。

> 目录约定：以下命令默认在工作空间根目录 `~/smartwheel` 下执行。

---

## 1. GUI 运行指令

SmartWheel 默认通过 systemd 用户服务 `smartwheel.service` 开机自动启动整套
ROS 后端（`full_system.launch.py`），其中已经包含 Web UI（端口 8080）。
原生桌面 GUI 是另一个独立进程，需要时再起。

### 1.1 启动 / 停止 / 查看后台服务

注意：start 只启动这一次；enable 是设置开机自启。两者独立。
  
  ┌──────────────┬───────────────────────────────────────────────────────┐
  │ 操作         │ 命令                                                  │
  ├──────────────┼───────────────────────────────────────────────────────┤
  │ 启动一次     │ systemctl --user start smartwheel.service             │
  ├──────────────┼───────────────────────────────────────────────────────┤
  │ 停止一次     │ systemctl --user stop smartwheel.service              │
  ├──────────────┼───────────────────────────────────────────────────────┤
  │ 重启         │ systemctl --user restart smartwheel.service           │
  ├──────────────┼───────────────────────────────────────────────────────┤
  │ 设置开机自启 │ systemctl --user enable smartwheel.service            │
  ├──────────────┼───────────────────────────────────────────────────────┤
  │ 取消开机自启 │ systemctl --user disable smartwheel.service           │
  ├──────────────┼───────────────────────────────────────────────────────┤
  │ 查状态       │ systemctl --user status smartwheel.service --no-pager │
  └──────────────┴───────────────────────────────────────────────────────┘
  
  enable 后下次开机会自动 start；disable 后下次开机不会自动 start，但当前已经
  start 的进程不会被立刻停掉。

```bash
# 启动 ROS 全栈（含 Web UI、传感器、定位、导航）
systemctl --user start smartwheel.service

# 停止 ROS 全栈（雷达 SDK 会被通知 stop，扫描停止）
systemctl --user stop smartwheel.service

# 查看运行状态
systemctl --user status smartwheel.service --no-pager

# 重启（修改配置或代码后让节点重新加载）
systemctl --user restart smartwheel.service

# 临时关闭开机自启（不会立刻停止当前进程）
systemctl --user disable smartwheel.service

# 恢复开机自启
systemctl --user enable smartwheel.service
```

启动后等待 5-20 秒让所有节点初始化完成，然后用以下方式访问：

- **Web UI**：浏览器打开 `http://<orin_ip>:8080`，本机直接访问 `http://localhost:8080`
- **原生桌面 GUI**：见 1.2

### 1.2 启动原生桌面 GUI

桌面 GUI 是独立进程，**前置条件：`smartwheel.service` 已经在运行**（否则
GUI 没有 ROS 数据可显示）。

```bash
cd ~/smartwheel
bash scripts/run_native_gui.sh
```

关闭桌面 GUI：直接关闭窗口或在终端按 `Ctrl+C`。

### 1.3 不使用自启服务，纯手动启动

如果要在调试期间完全控制 ROS 启动：

```bash
# 1. 关闭自启服务，避免抢占
systemctl --user stop smartwheel.service

# 2. 手动启动 ROS 全栈（前台运行，Ctrl+C 退出）
cd ~/smartwheel
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch wheelchair_bringup full_system.launch.py

# 或者只跑传感器，不跑 Nav2
ros2 launch wheelchair_bringup sensors.launch.py mode:=real
```

## 2. 雷达开关（按需启用 XT-M60 扫描）

为了避免 ROS 自启时雷达持续全速扫描发热，**默认情况下 `smartwheel.service`
启动时不启动 XT-M60 适配器**——雷达通电待机（可 ping 通），但不发激光、不出
点云。当需要建图、定位、自动导航时再临时启用。

启用与否由环境变量 `SMARTWHEEL_ENABLE_XTM60` 控制（`true` / `false`）。

### 2.1 检查当前是否在扫描

```bash
# 看适配器节点是否在跑
pgrep -af xtm60_adapter_node | grep -v grep || echo "(no adapter, radar idle)"

# 看 right 雷达是否有点云输出
source /opt/ros/humble/setup.bash
source ~/smartwheel/install/setup.bash
timeout 4 ros2 topic hz /xtm60/right/points 2>&1 | tail -3
```

- `(no adapter, radar idle)` + topic 无数据 → 雷达 A 状态（待机，不发热）
- 看到 `average rate: ~10 Hz` → 雷达 C 状态（扫描中，主要发热源）

### 2.2 临时启用雷达（一次性，不影响开机自启默认值）

```bash
# 1. 在 service 上加一次性 override
systemctl --user edit smartwheel.service
# 在打开的编辑器里写：
#   [Service]
#   Environment=SMARTWHEEL_ENABLE_XTM60=true
# 保存退出

# 2. 重新加载并重启
systemctl --user daemon-reload
systemctl --user restart smartwheel.service

# 3. 等 8-15 秒，验证
sleep 12
ros2 topic hz /xtm60/right/points
```

用完想恢复默认：再次 `systemctl --user edit smartwheel.service`，把那行改成
`Environment=SMARTWHEEL_ENABLE_XTM60=false`（或直接清空文件），再
`daemon-reload + restart`。

### 2.3 临时启用雷达（不通过 service，仅当前 shell 生效）

适合调试时临时拉起一次：

```bash
# 1. 停掉自启 service
systemctl --user stop smartwheel.service

# 2. 当前 shell 设置环境变量后手动跑
cd ~/smartwheel
source /opt/ros/humble/setup.bash
source install/setup.bash
SMARTWHEEL_ENABLE_XTM60=true bash scripts/autostart_smartwheel.sh
```

`Ctrl+C` 退出后雷达 SDK 会被通知 `stop()`，回到 A 状态。

### 2.4 完全不通过 service，直接 launch

最直接的"用完就关"方式，跳过 systemd：

```bash
systemctl --user stop smartwheel.service
cd ~/smartwheel
source /opt/ros/humble/setup.bash
source install/setup.bash

# 启用雷达
ros2 launch wheelchair_bringup full_system.launch.py enable_xtm60_radar:=true

# 不启用雷达（默认就是 false，可不写）
ros2 launch wheelchair_bringup full_system.launch.py enable_xtm60_radar:=false
```

`Ctrl+C` 退出后所有节点都会被 destroy_node 清掉。

### 2.5 关于建图入口

`mapping.launch.py` 是另一条独立的 launch，**它不受 `enable_xtm60_radar` 控制
**——建图本身就需要雷达，所以它会无条件启动 XT-M60 适配器：

```bash
# 建图时直接用这个，不需要管 SMARTWHEEL_ENABLE_XTM60
ros2 launch wheelchair_bringup mapping.launch.py use_mock:=false
```

注意：跑 `mapping.launch.py` 之前要先 `systemctl --user stop smartwheel.service`，
避免节点同名冲突。

### 2.6 几个常见问题

- **改了 `Environment=` 但 restart 没生效**：必须先 `systemctl --user daemon-reload`
  再 `restart`。
- **Web UI 显示雷达离线、安全状态变 STOP**：这是预期。雷达没启时
  `safety_supervisor_node` 检测到 `/scan` 离线 → `/system_stop_required=true` →
  `/cmd_vel_safe=0`。所以默认状态下 Nav2 也驱不动底盘。要恢复自动驾驶就启用雷达。
- **怎么确认默认是不开的**：`systemctl --user show smartwheel.service -p Environment | grep XTM60`，应该看到 `SMARTWHEEL_ENABLE_XTM60=false`。

## 3. WSAD 操控轮椅（仅离地空转测试用）


> **⚠️ 此脚本仅用于硬件 bring-up 测试和电机方向标定。它直接通过 Modbus RTU
> 写入 ZLAC8030 寄存器，绕过 `safety_supervisor_node` 的全部安全约束。
> 仅在轮子离地、周围无人、有人随时可按硬件急停的前提下使用。**

### 3.1 前置条件检查清单

执行命令前必须满足全部条件：

1. **驱动轮已离地**（垫高、车架支起）
2. **`smartwheel.service` 已停止**——避免它跟脚本同时写 ZLAC8030 串口造成冲突
3. **物理急停按钮在手边**
4. **使用稳定软链而不是 ttyACMx**（重要！）：USB 枚举顺序每次开机可能变，
   `/dev/ttyACM0` 这次是 IMU、下次可能是 ZLAC8030，硬编码会出错。udev 规则
   按 USB Product ID 给设备建了稳定软链：

   - `/dev/smartwheel_zlac8030` → ZLAC8030/KeepLINK（**WSAD 脚本要用这个**）
   - `/dev/smartwheel_h30_imu`  → H30 IMU
   - `/dev/smartwheel_ultrasonic` → FD07-34 超声波（在 ttyUSB1）

   脚本默认 `--port /dev/ttyACM0` 不可靠（那个编号会随重启换设备），
   **必须显式传 `--port /dev/smartwheel_zlac8030`**。

### 3.2 启动 WSAD 测试

```bash
# 1. 停止后台 ROS，释放 ZLAC8030 串口
systemctl --user stop smartwheel.service

# 2. 启动 WSAD 控制（必须传安全确认参数）
cd ~/smartwheel
python3 scripts/jog_zlac8030d_motor.py \
  --port /dev/smartwheel_zlac8030 \
  --rpm 20 \
  --i-confirm-wheels-off-ground
```

参数说明：

- `--port /dev/smartwheel_zlac8030`：ZLAC8030 串口（必填，默认值不适用本机）
- `--rpm 20`：单次按键的 RPM，允许 1–60，默认 20
- `--i-confirm-wheels-off-ground`：你已确认轮子离地，必填
- `--baud 115200`：波特率，默认 115200，无需修改
- `--deadman-sec 0.35`：松手多久自动归零，默认 0.35 秒
- `--invert-right` / `--no-invert-right`：右轮反向，默认开启（与 zlac8030_base.yaml 一致）

### 3.3 操作按键

| 按键 | 动作 |
| --- | --- |
| `w` | 前进（按下时持续，松开 0.35 秒自动归零） |
| `s` | 后退 |
| `a` | 左转（差速：左轮反、右轮正） |
| `d` | 右转 |
| `x` | 立刻零速（不锁定） |
| `空格` 或 `e` | **软件急停**（锁定，需按 `r` 才能恢复） |
| `r` | 急停后重新使能驱动器 |
| `q` 或 `Ctrl+C` | 退出脚本（退出前会自动急停） |

脚本每秒打印一次寄存器快照（`status: ...`），方便观察电机状态。

### 3.4 测试结束

```bash
# 退出脚本（按 q 或 Ctrl+C）

# 恢复 ROS 后台服务
systemctl --user start smartwheel.service
```

### 3.5 常见问题

- **`could not open port /dev/smartwheel_zlac8030`**：检查 USB 是否插好、`smartwheel.service`
  是否已停（占用串口）、当前用户是否在 `dialout` 组（`groups | grep dialout`）。
- **方向反了**：用 `--forward-sign -1`（默认）或 `--forward-sign 1` 切换前进方向；
  用 `--turn-sign 1` / `--turn-sign -1` 切换 A/D 转向方向。
- **Refusing jog mode without --i-confirm-wheels-off-ground**：必须显式加这个
  参数才能进入 jog 模式，是有意为之的安全门槛，不要去掉。
- **运动一会儿就停**：deadman 机制工作正常——松手 0.35 秒自动归零，按住对应键
  才会持续运动。

## 附录：紧急情况

- **物理急停**：按下方向盘/扶手上的硬件急停按钮（连接到 `/emergency_stop_hw`）
- **软件急停（命令行）**：

  ```bash
  ros2 topic pub --once /emergency_stop_sw std_msgs/msg/Bool "{data: true}"
  ```

- **强制零速**：

  ```bash
  ros2 topic pub -r 10 /cmd_vel_safe geometry_msgs/msg/Twist \
    "{linear: {x: 0.0}, angular: {z: 0.0}}"
  ```

- **整体下电**：拍硬件急停 + 关掉电源开关
